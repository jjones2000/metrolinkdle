# Metrolinkdle

Manchester Metrolink Wordle — React frontend on Netlify + FastAPI backend on Render.

```
metrolinkdle/
├── backend/                 ← FastAPI (Python)
│   ├── main.py              ← API routes, daily target, stats
│   ├── stops.json           ← stop names (generated by build_graph.py)
│   ├── requirements.txt
│   ├── Procfile
│   └── .env.example
│
├── frontend/                ← React + Vite
│   ├── src/
│   │   ├── main.jsx
│   │   ├── App.jsx          ← root component, game state
│   │   ├── api.js           ← all fetch() calls in one place
│   │   ├── gameLogic.js     ← BFS, clue evaluation, share text (pure JS)
│   │   ├── networkData.js   ← full TfGM graph embedded (generated)
│   │   ├── hooks/
│   │   │   ├── usePlayerId.js
│   │   │   ├── useDaily.js
│   │   │   └── useStats.js
│   │   └── components/
│   │       ├── NetworkMap.jsx
│   │       ├── StopSearch.jsx
│   │       ├── GuessRow.jsx
│   │       ├── ResultModal.jsx
│   │       └── HowToModal.jsx
│   ├── index.html
│   ├── vite.config.js       ← proxies /api to localhost:8000 in dev
│   ├── .env.development
│   └── .env.production      ← set VITE_API_URL to your Render URL
│
├── render.yaml              ← Render deployment config
├── build_graph.py           ← regenerates networkData.js + stops.json
├── Metrolink_Lines_Functional.json
├── Metrolink_Stops_Functional.json
├── metrolink_graph.json
├── dev.sh                   ← starts both servers locally
└── README.md
```

---

## Architecture

```
Browser (Netlify)                   FastAPI (Render)
─────────────────────               ───────────────
App.jsx
  └─ useDaily()    ──── GET /api/daily ────▶  deterministic daily stop
  └─ useStats()    ──── GET /api/stats/:id ──▶  player stats from SQLite
                   ──── POST /api/stats ────▶  record completed game

All graph/BFS logic stays client-side — no round-trips for guesses.
```

## Local Development

Run both servers at once:

```bash
./dev.sh
```

Or manually:

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
uvicorn main:app --reload
# http://localhost:8000  |  docs: http://localhost:8000/docs
```

### Frontend
```bash
cd frontend
npm install
npm run dev
# http://localhost:5173  (/api proxied to localhost:8000)
```

## Deploy — Render (Backend)

1. Push your code to GitHub

2. Go to [render.com](https://render.com) → **New → Web Service**

3. Connect your GitHub repo, then set:
   - **Root Directory:** `backend`
   - **Runtime:** Python
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

4. Select the **Free** plan

5. Under **Environment**, add:
   ```
   ALLOWED_ORIGINS=https://your-app.netlify.app
   DATA_DIR=/opt/render/project/data
   ```

6. Click **Deploy** — your backend URL will be `https://metrolinkdle-backend.onrender.com`

> **Note:** On the free tier Render sleeps after 15 minutes of inactivity. The first request after a quiet period takes ~30 seconds to wake up. For a daily game this is usually fine.

---

## Deploy — Netlify (Frontend)

1. Go to [netlify.com](https://netlify.com) → **Add new site → Import from Git**

2. Set:
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `frontend/dist`

3. **Site configuration → Environment variables**, add:
   ```
   VITE_API_URL=https://metrolinkdle-backend.onrender.com
   ```

4. Click **Deploy** — your app will be live at `your-app.netlify.app`

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/health` | Health check |
| `GET`  | `/api/daily` | Today's target stop |
| `GET`  | `/api/daily/{YYYY-MM-DD}` | Target for a specific date (testing) |
| `POST` | `/api/stats` | Record a completed game |
| `GET`  | `/api/stats/{player_id}` | Get stats for a player |

Interactive docs: `http://localhost:8000/docs`
