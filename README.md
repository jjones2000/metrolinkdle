## Architecture

```
Browser (Vercel)                    FastAPI (Railway)
в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ               в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
App.jsx
  в””в”Ђ useDaily()    в”Ђв”Ђв”Ђв”Ђ GET /api/daily в”Ђв”Ђв”Ђв”Ђв–¶  deterministic daily stop
  в””в”Ђ useStats()    в”Ђв”Ђв”Ђв”Ђ GET /api/stats/:id в”Ђв”Ђв–¶  player stats from SQLite
                   в”Ђв”Ђв”Ђв”Ђ POST /api/stats в”Ђв”Ђв”Ђв”Ђв–¶  record completed game

All graph/BFS logic stays client-side вЂ” no round-trips for guesses.
```

**Why this split?**
- The daily target must be consistent across all clients в†’ server decides it
- Stats need persistence across devices/browsers в†’ server stores them
- BFS/graph stays in the browser в†’ fast, no latency, no server load

## Local Development

### 1. Backend

```bash
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
uvicorn main:app --reload
# API running at http://localhost:8000
# Docs at http://localhost:8000/docs
```

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
# App running at http://localhost:5173
# /api calls are proxied to localhost:8000 by vite.config.js
```

## Updating Network Data

If TfGM releases updated GeoJSON:

```bash
# From the project root
python build_graph.py \
  --stops Metrolink_Stops_Functional.json \
  --lines Metrolink_Lines_Functional.json

# Then copy outputs to the right places:
#   metrolink_graph.json в†’ embed in frontend/src/networkData.js
#   stop names           в†’ backend/stops.json
```

Or run the included helper:

```bash
python build_graph.py && python scripts/update_data.py
```

## Deploy to Railway (Backend)

1. Push `backend/` to a GitHub repo (or the whole monorepo)
2. New project в†’ **Deploy from GitHub repo** в†’ select the backend folder
3. Railway auto-detects Python via Nixpacks
4. **Add a Volume**: mount path `/data` (matches `railway.toml`)
5. Set environment variables:
   ```
   ALLOWED_ORIGINS=https://your-app.vercel.app
   DATA_DIR=/data
   ```
6. Copy the Railway public URL (e.g. `https://metrodle-backend.up.railway.app`)

## Deploy to Vercel (Frontend)

1. Push `frontend/` to GitHub
2. New project on Vercel в†’ import repo
3. Framework preset: **Vite**
4. Add environment variable:
   ```
   VITE_API_URL=https://metrodle-backend.up.railway.app
   ```
5. Deploy вЂ” done.

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/health` | Railway health check |
| `GET`  | `/api/daily` | Today's target stop `{stop, date}` |
| `GET`  | `/api/daily/{YYYY-MM-DD}` | Target for any date (testing) |
| `POST` | `/api/stats` | Record a completed game |
| `GET`  | `/api/stats/{player_id}` | Get aggregated stats for a player |

Interactive docs: `http://localhost:8000/docs`
