## Architecture

```
Browser (Netlify)                   FastAPI (Render)
─────────────────────               ────────────────
App.jsx
  └─ useDaily()    ──── GET /api/daily ────▶  daily stop + game number
  └─ useStats()    ──── GET /api/stats/:id ──▶  player stats from SQLite
                   ──── POST /api/stats ────▶  record completed game

All graph/BFS logic stays client-side — no round-trips for guesses.
```

**Why this split?**
- The daily target must be consistent across all clients → server decides it
- Game number is calculated server-side from days since launch date
- Stats need persistence across devices/browsers → server stores them in SQLite
- BFS/graph stays in the browser → fast, no latency, no server load

## Local Development

Run both servers at once:

```bash
./dev.sh
```

Or manually:

### Backend
```bash
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
uvicorn main:app --reload
# http://localhost:8000  |  docs: http://localhost:8000/docs
```

### Frontend
```bash
cd frontend
npm install
npm run dev
# http://localhost:5173  (/api proxied to localhost:8000)
```

## Deploy — Render (Backend)

1. Push your code to GitHub

2. Go to [render.com](https://render.com) → **New → Web Service**

3. Connect your GitHub repo, then set:
   - **Root Directory:** `backend`
   - **Runtime:** Python
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`

4. Select the **Free** plan

5. Under **Environment**, add:
   ```
   ALLOWED_ORIGINS=https://your-app.netlify.app
   DATA_DIR=/opt/render/project/data
   ```

6. Click **Deploy** — your backend URL will be `https://metrolinkdle-backend.onrender.com`

> **Note:** On the free tier Render sleeps after 15 minutes of inactivity. The first request after a quiet period takes ~30 seconds to wake up. Use [cron-job.org](https://cron-job.org) to ping `/health` every 14 minutes to keep it awake for free.

---

## Deploy — Netlify (Frontend)

1. Go to [netlify.com](https://netlify.com) → **Add new site → Import from Git**

2. Set:
   - **Base directory:** `frontend`
   - **Build command:** `npm run build`
   - **Publish directory:** `frontend/dist`

3. **Site configuration → Environment variables**, add:
   ```
   VITE_API_URL=https://metrolinkdle-backend.onrender.com
   ```

4. Click **Deploy** — your app will be live at `your-app.netlify.app`

## Game Number

The puzzle number is calculated in `backend/main.py` from days elapsed since `LAUNCH_DATE`. To change when #1 is, update:

```python
LAUNCH_DATE = date(2026, 3, 13)  # Puzzle #1
```

## API Reference

| Method | Path | Description |
|--------|------|-------------|
| `GET`  | `/health` | Health check |
| `GET`  | `/api/daily` | Today's stop, date, and game number |
| `GET`  | `/api/daily/{YYYY-MM-DD}` | Target for a specific date (testing) |
| `POST` | `/api/stats` | Record a completed game |
| `GET`  | `/api/stats/{player_id}` | Get stats for a player |

Interactive docs: `http://localhost:8000/docs`

## Updating Network Data

If TfGM releases updated GeoJSON:

```bash
python build_graph.py \
  --stops Metrolink_Stops_Functional.json \
  --lines Metrolink_Lines_Functional.json
```
